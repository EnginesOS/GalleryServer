Technical brief

Engines is a managed system of containerised applications and services. It is controlled through a fully-featured web front end. Its purpose is to provide nontechnical users with tools to easily launch a machine (VM or BM) and install a collection of server software. Software can be easily configured for data access and sharing.

Engines provides users with the benefits of self-hosting, such as data sovereingty, without the challenges normally associated with DIY servers.

Engines can run a wide range of software while maintaining system and data security. Application images are built locally. This underpins security and makes it possibe to automatically integrate software with a local environment and system services.

Software blueprints are used to define an installation and to configure the associated image build process. The installation process is generally fully-automaed. Some installation processes complete after receiving input from the user via a web iterface. Users find and access blueprints for new software in software galleries.

Engines is suited to personal and small- to medium-scale organisational use. Individuals can use Engines to host a blog, media library, private instant message service, social networking feed, multiplayer game servers, and an electronic portfolio. Commercial users could run software for an online store, project management, CRM, fleet tracking, voip server, accounting, document sharing, group contacts/calendar and ERP. Schools could host an education platform server, mail and more. There is an almost limitless range of software applications that could be blueprinted to run on Engines.

Engines is currently in Alpha on a weekly release cycle with a planned Beta Release due in May. The software is released under the Apache 2.0 license.
